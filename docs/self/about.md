# Что такое onbo

`onbo` — опенсорс-ассистент онбординга «под любой софт». Он принимает обращения
пользователя любым каналом (Telegram, веб-чат, голос), понимает их и либо
отвечает по базе знаний (RAG с учётом роли/отдела), либо выполняет действие над
профилем (сменить язык, email и т.п.).

Этот файл индексируется в публичную коллекцию `about`, поэтому вопрос «как тебя
настроить?» ассистент отвечает сам, обычным путём поиска.

## Как настроить

1. Скопируйте `config/settings.yaml` и задайте переменные окружения
   (`OPENAI_API_KEY`, `POSTGRES_DSN`, `REDIS_URL`, `TELEGRAM_BOT_TOKEN`).
2. Опишите действия вашего продукта в `config/actions.yaml`.
3. Наполните базу знаний: `onbo kb add-doc ./handbook --collection support` и
   `onbo kb seed`.
4. Проиндексируйте доки о самом ассистенте: `onbo about`.
5. Запустите канал: `onbo serve telegram` или `onbo serve web`.

## Режимы действий

- `chat` — низкий риск, выполняется сразу (например, смена языка).
- `confirm` — важное, но обратимое: спрашивает «Ок/Отмена» (например, email).
- `link` — чувствительные данные (пароль, телефон): в чате не трогаем, отдаём ссылку.

## Как добавить плагин

- Новый канал → новый файл в `onbo/channels/` по интерфейсу `Channel`.
- Новое действие → запись в `config/actions.yaml` + файл в `onbo/handlers/actions/`.
- Новый источник для базы знаний → файл в `onbo/kb/sources/`.

Ядро (`onbo/core/`) при этом не меняется.
