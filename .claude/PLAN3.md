Статус: **не выполняется**, только зафиксировано. База — `.claude/PLAN.md` (шаги 1–12)

## Контекст

Текущая админка (`onbo/channels/admin_api.py`, константа `_ADMIN_HTML`) — это одна
страница с формой и таблицей, целиком отрисованная на сервере. Годится для демо на
десяток Q&A-пар и разваливается на реальном объёме: пары, документы и коллекции
исчисляются сотнями и тысячами. Плюс `/admin` и `/admin/api/*` смонтированы без
какой-либо аутентификации — кто угодно с доступом к порту правит базу знаний.

## 20. Доступ к админке (первый по важности)

- Аутентификация на весь префикс `/admin` и `/admin/api/*`: логин+пароль или токен
  из окружения (`ADMIN_TOKEN`) для одиночного деплоя, OIDC/заголовок от обратного
  прокси — для корпоративного. Отдельный флаг `features.admin_auth`.
- Роли: кто читает КБ, кто правит, кто запускает переиндексацию. Сейчас любой
  вызов `POST /admin/api/reindex` кладёт индекс.
- CSRF-токен на мутирующие формы, rate limit на запись, аудит изменений
  (кто/когда/что правил — таблица рядом с `action_log`).
- В README — явное предупреждение: без этого `/admin` наружу не выставлять.

## 21. Постраничность и поиск вместо одной таблицы

- `GET /admin/api/qa` и `/documents`: параметры `limit`/`offset` (или курсор),
  `q` (полнотекстовый поиск по вопросу/ответу), фильтры по коллекции, отделу,
  ролям, наличию видео; сортировка по дате изменения.
- Ответ отдаёт `total`, чтобы UI рисовал постраничную навигацию.
- Индексы в Postgres под фильтры и поиск (`kb_qa(collection)`, GIN по тексту).
- UI: таблица с пагинацией, строкой поиска и фильтрами; серверная отдача страницами,
  без выгрузки всей базы в браузер.

## 22. Разделение на страницы и модальные окна

- Разнести одну простыню на маршруты: `/admin` (сводка), `/admin/qa`,
  `/admin/qa/{id}`, `/admin/collections`, `/admin/documents`, `/admin/documents/{id}`.
- Создание и правка Q&A — в модальном окне или на своей странице, с валидацией и
  предпросмотром того, как пара попадёт в индекс (чанки, коллекция, доступ).
- Массовые операции: выделение нескольких строк → удалить, сменить коллекцию,
  переиндексировать выборочно.
- Импорт/экспорт YAML прямо из UI (сейчас только CLI `onbo kb import`).

## 23. Работа с медиа

- Загрузка mp4 файлом в `media/kb/` через админку вместо ручной вставки URL в поле
  «Видео»: `POST /admin/api/media` → сохранение + подстановка `video_url`.
- Превью ролика в карточке пары, замена и удаление файла.
- Ограничение размера и типа, чтобы каталог не превращался в свалку.

## 24. Чат-UI на `/`

Сейчас у веб-канала нет корневой страницы: `GET /` отдаёт 404, чат доступен только
как `POST /chat`. Нужна простая страница чата (история сообщений, кнопка записи
голоса под `POST /voice`, подтверждения действий под `POST /confirm`), чтобы
продукт можно было показать без curl.

## 25. Наблюдаемость

- Статус индексации: сколько пар в Postgres против точек в Qdrant, когда была
  последняя переиндексация, какие пары не проиндексированы.
- Журнал действий пользователей (`action_log`) в UI — что ассистент выполнил,
  что упало.
