# Voice reference — образец голоса для клонирования

Положи сюда файл `voice-ref.wav` — 30–60 секундную запись голоса, который
ElevenLabs скопирует и будет озвучивать видео базы знаний. Имя файла **обязательно**
`voice-ref.wav` — `kb_pipeline.py` ищет именно его.

## Требования к записи

- **Длительность:** 30–60 секунд (короче — голос звучит роботом, длиннее — лишний шум).
- **Формат:** WAV, моно, 48 kHz, 16-bit (`.wav`). mp3 тоже примет, но WAV стабильнее.
- **Тишина:** тихая комната, без вентиляторов/кондиционера/улицы.
- **Микрофон:** USB-микрофон или наушники с хорошим миком. Встроенный мик ноутбука — рискованно.
- **Текст:** живая осмысленная речь с интонацией — вопросы, утверждения, паузы.
  **Не** считай «раз-два-три», **не** мычи — модели нужен естественный паттерн речи.
- **Уровень:** говори как обычно; проверь, что пик не клиппится.
- Без музыки и эффектов на фоне.

## Как записать (Linux)

**Через `arecord` + `ffmpeg`:**

```bash
arecord -f cd -d 45 raw.wav                 # ~45 сек с дефолтного входа
ffmpeg -i raw.wav -ar 48000 -ac 1 voice-ref.wav
```

**Или в Audacity:** запиши ~45 сек → `File → Export → Export as WAV` →
48000 Hz, mono, 16-bit PCM → сохрани как `voice-ref.wav` в эту папку.

**Из телефонной записи:** перекинь `.m4a`/`.mp3` на компьютер и сконвертируй:

```bash
ffmpeg -i memo.m4a -ar 48000 -ac 1 voice-ref.wav
```

## Куда положить

```
.claude/skills/kb-video/refs/voice-ref.wav
```

Файл в `.gitignore` — в репозиторий не попадёт.

## Чей голос можно использовать

- **Свой** — без вопросов.
- **Чужой** — только с явного разрешения этого человека.
- **Известных людей / актёров** — нет. ElevenLabs детектит и блокирует; это плохая идея
  и юридически, и морально.
